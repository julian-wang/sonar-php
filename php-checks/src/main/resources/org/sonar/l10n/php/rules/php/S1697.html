无论是空检测中的等于操作符还是下面相反的逻辑操作符，解引用之前的空检测代码只是看起来安全的。不幸的是其效果恰恰相反，对象为空-测试然后当且仅当它为空解引用，导致了确保空指针时才解引用。

不规范的代码示例

if ($obj == null && $obj->isOpen()) {
  echo "Object is open";
}

if ($obj != null || $obj->isOpen()) {
  echo "Object is not open";
}

规范的解决方案

if ($obj == null || $obj->isOpen()) {
  echo "Object is open";
}

if ($obj != null && !$obj->isOpen()) {
  echo "Object is not open";
}
