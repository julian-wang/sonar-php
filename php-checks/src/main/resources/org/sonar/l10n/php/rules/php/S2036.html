定义例如类、变量等符号的文件，可能会被多个文件引入。简单地说，除了声明新的符号以外，不应该对这些文件有任何影响。例如，一个包含类定义的文件不应该包含类似打印语句的副作用，这会在包含时自动评估。逻辑应该拆分成只包含符号定义的文件和只产生副作用的文件。符号定义文件中不允许的操作类型包括但不限于：

  • 产生输出
  • 修改ini设置
  • 产生错误和异常
  • 修改全局或静态变量
  • 读写文件

不规范的代码示例

<?php

print "Include worked!";

class foo {
  // ...
}

规范的解决方案

<?php

class foo {

  public function log() {
    print "Include worked!";
  }

}
