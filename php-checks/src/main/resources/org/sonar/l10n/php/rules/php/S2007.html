共享编码约定使得团队合作有效。当约定面向对象编程时，定义和使用全局变量或全局函数会造成混淆，而且难以正确使用，多种原因如下：

  • 冒着命名冲突的风险．
  • 全局函数必须是无状态的，否则他们会导致难以跟踪的错误．
  • 全局变量可以在任何地方更新，可能就不再是你期望的值了．
  • 很难用全局函数正确的测试类．

这些变量和函数应该移到一个类中，而不是声明为全局，可能需要标记成static的，以便不生成类实例而直接使用他们．

这条规则检测是否只是用面向对象编程，而且没有函数或过程在类外声明．

不规范的代码示例

<?php

$name = "Bob"; // Noncompliant

function doSomething($arg) {   // Noncompliant
  //...
}

class MyClass {
    //...
}

规范的解决方案

<?php 
class MyClass {

  public static $name = "Bob"; // Compliant

  public static function doSomething($arg) {              // Compliant
    //...
  }
  //...
}
