重写方法如果只是调用父类，而没有其他任何操作，是没有意义而且容易误导的。唯一合理的一种情况是，将重写函数声明为final，其效果等于锁定其父类行为。这条规则会忽略equals, hashCode和toString函数。

不规范的代码示例

class Child extends Parent {

  public function func($n,$m) {
    parent::func($n$m);  // Noncompliant
  }
}

class Parent {
  public function func($n, $m) {
    // do something
  }
}

规范的解决方案

class Child extends Parent {

  public function func($n,$m) {
    parent::func($n$m);
    // do additional things...
  }
}

class Parent {
  public function func($n, $m) {
    // do something
  }
}

或者

class Child extends Parent {
  // function eliminated
}

class Parent {
  public function func($n, $m) {
    // do something
  }
}
