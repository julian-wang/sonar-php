如果你抛出一个通用异常，例如 ErrorException, RuntimeException，或者库异常、框架异常，这将强制消费者捕获所有异常，包括他们也不知道如何处理的未知异常.

相反的，你应该抛出一个PHP标准库里已存在的子类型，或者创建继承自Exception的你自己的类型。

不规范的代码示例

throw new Exception();  // Noncompliant

规范的解决方案

throw new InvalidArgumentException();
// or
throw new UnexpectedValueException();

例外

在函数重写的签名中的通用异常将被忽略。 

@Override
public void myMethod() throws Exception {...}
