此规则验证单行注释不应放于代码行的结尾。这条规则背后的主要思想是为了保证可读性，尾随的注释会被正确的写入和格式化（正确对齐，不对代码视觉结构产生干扰，不至于太长而不可见），但大多数时候，代码自动格式化并不能正确处理，最终代码不具有可读性。注释最好放在代码前面的空行，这样他们将始终可见并可以被正确的格式化。

不规范的代码示例

$a = $b + $c; // This is a trailing comment that can be very very long

规范的解决方案

// This very long comment is better placed before the line of code
$a = $b + $c;

例外

默认情况下，"legalTrailingCommentPattern"属性允许忽略只包含一个词的注释：

doSomething(); //FIXME
