exit(...) 和 die(...)语句不应当在PHP网站页面中使用，这会导致非常糟糕的用户体验。在这种情况下，最终用户可能有这样的感觉：该网站宕机了或者遇到致命错误了。

当然，也可以使用PHP开发命令行程序，在这种情况下使用exit(...)或die(...)语句是合理的，但必须做出限制，不至于遍及整个程序。我们希望处理错误时使用异常，这些异常应该在离开程序前被捕获，在exit(...) 或 die(...)语句的帮助下明确退出代码。

不规范的代码示例

class Foo {
    public function bar($param)  {
        if ($param === 42) {
            exit(23);
        }
    }
}

规范的解决方案

class Foo {
    public function bar($param)  {
        if ($param === 42) {
            throw new Exception('Value 42 is not expected.');
        }
    }
}
