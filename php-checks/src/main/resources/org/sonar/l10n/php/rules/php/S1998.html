传递一个引用给函数参数意味着函数所做的任何修改都会影响原值，因为引用是将两变量指到了同一块内存。这个特性很难正确的使用，特别是调用者不期待这是一个引用，而函数调用的不正确使用会使代码更加低效而不是高效。

此外，根据PHP手册：

在PHP 5.3.0中，你会得到一个"call-time pass-by-reference"的警告，这已经过时。在PHP 5.4.0中，调用时引用传递已经被去掉了，这样会导致一个致命错误。

不规范的代码示例

myfun(&$name);  // Noncompliant

规范的解决方案

myfun($name);
