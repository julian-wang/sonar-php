PHP 4中用var关键字声明变量的方法已经过时在PHP 5的早期版本。尽管在最近的版本中不再认为过时，但是最好的实际仍然是不使用它。当 var出现时，它被解释为并当作public的同义词。因此，应该使用public替代。

来自PHP官方手册:

为了兼容性，PHP 4使用var关键字声明变量的方法仍然是支持的（看作public关键字的同义词）。在PHP 5.1.3之前的PHP 5中，它的使用会产生一个E_STRICT的警告。

不规范的代码示例

<?php
class Foo
{
    var $bar = 1;
}

规范的解决方案

<?php
class Foo
{
    public $bar = 1;
}
