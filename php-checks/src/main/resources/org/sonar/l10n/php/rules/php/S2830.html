依赖注入是一种软件设计模式，一个或多个依赖被注入或通过引用传递给依赖对象（或称客户端），然后变成了客户端状态的一部分。这种模式将客户端依赖的创建与其自身的行为分离，这可以使程序设计松耦合，遵循了依赖反转和单一责任原则。

不规范的代码示例

class SomeClass {

  public function __construct() {
    $this->object = new SomeOtherClass();
  }
}

规范的解决方案

class SomeClass {

  public function __construct(SomeOtherClass $object) {
    $this->object = $object;
  }
}
